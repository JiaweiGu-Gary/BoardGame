# Cardia 游戏 UI 布局优化总结

## 改动概述

重新设计了 Cardia 游戏的 UI 布局，为两个玩家增加了各自的弃牌堆显示区域，并实现了特殊的堆叠效果。同时优化了中间对战区域的卡牌显示，使整体布局更加紧凑高效。

## 主要改动

### 1. 新增弃牌堆组件 (`src/games/cardia/ui/DiscardPile.tsx`)

创建了专门的弃牌堆组件，实现以下功能：

- **明牌显示**：所有弃牌都以明牌形式展示
- **堆叠效果**：从下往上堆叠，最新弃牌在最上层
- **显示规则**：
  - 最新弃牌：显示完整卡面（28px × 42px，与手牌/对战区一致）
  - 历史弃牌：只显示左侧三分之一（约 9px 宽度），展示数字部分
  - 每张历史卡片向右偏移 36px，形成扇形堆叠效果
- **交互功能**：
  - 点击任意卡牌可放大查看
  - 悬停时有视觉反馈（亮度提升/缩放）
- **状态标记**：
  - 印戒标记（黄色圆圈，右上角，4px × 4px）
  - 修正标记（蓝色圆圈，左上角，4px × 4px，🔧图标）
  - 持续标记（紫色圆圈，左下角，4px × 4px，🔄图标）

### 2. 优化中间对战区域卡牌显示

**布局优化**：
- **VS 指示器覆盖设计**：VS 图标覆盖在两张卡牌之上，使用绝对定位居中显示
- **条件显示**：只有双方都打出卡牌后才显示 VS 指示器
- **卡牌重叠布局**：我的卡牌使用负 margin（-mt-4）与对手卡牌部分重叠，节省垂直空间
- **紧凑间距**：遭遇对之间间距为 gap-4

**卡牌尺寸**：
- 对战区域：28px × 42px（紧凑显示）

**VS 指示器样式**：
- 背景：白色（bg-white）
- 边框：灰色边框（border border-gray-300）
- 文字：text-sm，紫色（text-purple-600）
- 影响力对比：text-xs，深灰色（text-gray-600）
- 层级：z-20（覆盖在卡牌之上）
- 尺寸：紧凑设计（px-2 py-0.5）

### 3. 优化手牌区域卡牌显示

**卡牌尺寸**：
- 手牌区域：20px × 30px（更紧凑，可显示更多手牌）

### 4. 重新设计 Board 布局 (`src/games/cardia/Board.tsx`)

#### 布局结构调整

**对手区域（顶部）**：
```
[弃牌堆] [对手信息栏（姓名、印戒、手牌数、牌库数）]
```

**中央战场区域**：
```
[遭遇序列 - 双方打出的卡牌对决]
```

**我的区域（底部）**：
```
[弃牌堆] [我的手牌和信息栏]
```

#### 具体改动

1. **对手区域**：
   - 添加了对手弃牌堆（左侧）
   - 对手信息栏移除了弃牌数量显示（改为独立弃牌堆）
   - 使用 `flex` 布局，弃牌堆固定宽度，信息栏自适应

2. **我的区域**：
   - 添加了我的弃牌堆（左侧）
   - 手牌区域和信息栏保持不变
   - 使用 `flex` 布局，弃牌堆固定宽度，手牌区域自适应

3. **PlayerInfoBar 组件**：
   - 移除了弃牌数量显示（`🗑️ 弃牌: X`）
   - 保留了手牌数量和牌库数量显示

### 5. 技术实现细节

#### 弃牌堆组件实现

```typescript
// 卡牌尺寸定义（与手牌/对战区一致）
const cardWidth = 28;       // 标准卡牌宽度
const cardHeight = 42;      // 标准卡牌高度（2:3 比例）

<div style={{
    left: `${index * offsetStep}px`,  // 每张卡片向右偏移 36px

    height: `${cardHeight}px`,
    zIndex: index,                    // 层级递增
}}>
    <div style={{ 
        width: `${cardWidth}px`, 
        height: `${cardHeight}px`
    }}>
        <OptimizedImage ... />
    </div>
</div>
```

## 用户体验改进

1. **信息可见性**：
   - 弃牌堆内容一目了然，无需点击查看
   - 弃牌堆扇形效果非常明显（36px 偏移），历史卡片清晰可辨
   - VS 指示器覆盖在卡牌之上，醒目且不占用额外空间
   - VS 指示器只在双方都打出卡牌后显示，避免干扰
   - 对战区域卡牌紧凑显示，节省空间

2. **空间利用**：
   - 弃牌堆采用紧凑的堆叠设计，扇形效果美观
   - 对战区域卡牌重叠布局（-mt-4），垂直空间利用率提升
   - VS 指示器覆盖设计，不占用额外垂直空间
   - 遭遇对间距缩小（gap-4），可显示更多遭遇对
   - 整体布局更紧凑，信息密度更高
   - 所有卡牌尺寸统一（除手牌更小），视觉一致性好

3. **交互友好**：
   - 点击任意卡牌可放大查看详情
   - 悬停时有明显的视觉反馈
   - VS 指示器有白色背景和边框，易于识别
   - VS 指示器条件显示，不会在等待阶段造成干扰

4. **视觉层次**：
   - 最新弃牌突出显示，历史卡片部分可见
   - 弃牌堆扇形堆叠效果非常明显，层次感强
   - VS 指示器使用 z-index 覆盖在卡牌之上，层次清晰
   - 对战区域卡牌重叠但不遮挡关键信息
   - 影响力对比清晰易读

5. **状态标记**：
   - 重要状态（印戒、修正、持续）直观显示
   - 标记尺寸适中（4px × 4px），不遮挡卡牌内容
   - 颜色区分明确（黄色=印戒，蓝色=修正，紫色=持续）

6. **视觉效果**：
   - VS 指示器使用白色背景，简洁清爽
   - 紫色文字与游戏主题一致
   - 阴影效果使 VS 指示器更突出

1. **响应式设计**：

## 总结

本次优化成功实现了 Cardia 游戏的弃牌堆可视化需求和整体布局优化：

1. **弃牌堆**：采用创新的堆叠显示方式（左侧三分之一），扇形偏移 36px，层次感非常强，卡牌尺寸与对战区域统一（28px × 42px），既节省了屏幕空间，又提供了良好的信息可见性
2. **对战区域**：
   - VS 指示器覆盖在两张卡牌之上，使用绝对定位和 z-index 实现层叠效果
   - VS 指示器改为白色背景，简洁清爽，紫色文字与游戏主题一致
   - VS 指示器只在双方都打出卡牌后显示，避免干扰
   - 卡牌重叠布局（-mt-4），垂直空间利用率大幅提升
   - 遭遇对间距缩小（gap-4），可以显示更多遭遇对
   - 整体布局更紧凑，视觉效果更好
3. **手牌区域**：卡牌尺寸调整为 20px × 30px，更紧凑，可以显示更多手牌
4. **视觉一致性**：弃牌堆、对战区域卡牌尺寸统一（28px × 42px），状态标记样式协调
5. **空间优化**：整体布局更紧凑，信息密度更高，可以在有限的屏幕空间内显示更多内容
6. **视觉效果**：VS 指示器使用白色背景和紫色文字，简洁现代，与游戏主题一致

这些改进不仅提升了游戏的视觉体验，也为玩家提供了更清晰的游戏信息展示，同时优化了空间利用，使得界面更加紧凑高效。VS 指示器的覆盖设计和条件显示是创新的布局方案，既节省了空间，又保持了视觉的醒目性，同时避免了不必要的干扰。
