## Requirements

### Requirement: Mobile-First 视口与安全区
项目 MUST 以移动端体验为主导，所有“对局页（play）”在主流移动设备尺寸下 MUST 满足：

- 核心游戏区（战场/公共区/信息条）在首屏可见
- 不依赖整页 `transform: scale()` 缩放来塞入内容
- 处理 iOS Safari 动态地址栏、刘海安全区（safe-area）等差异

#### Scenario: iPhone 竖屏首屏稳定
- **GIVEN** viewport 为 390x844（iPhone 14/15 Pro 常用）
- **WHEN** 进入任意游戏对局页
- **THEN** 页面不应出现明显的整页纵向滚动（允许极小误差）
- **AND** 核心游戏区与手牌区关键元素可见

#### Scenario: iPhone 横屏不应触发整页缩放
- **GIVEN** viewport 为 896x414（iPhone XR 横屏常用）
- **WHEN** 进入对局页并渲染完整战场与手牌
- **THEN** `#root` 不应被缩放（`transform` 不应使 scaleX < 0.99）

### Requirement: 根容器缩放开关（逐游戏）
项目 MUST 提供逐游戏的“移动端横屏根缩放”策略开关，并且默认应偏向 Mobile-First：

- 对局页在移动端横屏下，不应默认启用整页根缩放。
- 若某些旧游戏短期无法适配，可临时使用 `scale-root` 作为兼容路径，但必须在对应游戏的 manifest 中显式声明。

#### Scenario: 游戏声明 responsive 时不触发根缩放
- **GIVEN** 当前游戏 manifest 声明 `mobileLandscapeLayout = 'responsive'`
- **WHEN** 在移动端横屏进入对局页
- **THEN** `#root` 不应应用基于 `100vw/1280` 的缩放样式

### Requirement: 核心游戏区比例适配
对局页 MUST 通过“容器约束 + 内部滚动”实现适配：

- 优先保证核心交互区可操作（战场/行动按钮/关键提示）
- 次要信息（日志、帮助、装饰）允许折叠或延迟展示

#### Scenario: 核心区不被手牌区遮挡
- **GIVEN** 移动端竖屏
- **WHEN** 手牌区出现多张牌
- **THEN** 战场最底部的关键卡牌不应被手牌区明显遮挡

### Requirement: 触控热区与交互可达性
所有可点击元素（按钮/卡牌/选项）在移动端 MUST 满足：

- 可点击热区最小尺寸：44x44 CSS px（推荐 48x48）
- 允许通过“视觉尺寸 + invisible hit-area（如 padding/伪元素）”扩展热区
- 悬停（hover）效果不应作为唯一信息入口；移动端需可替代触发（点击/长按/按钮）

#### Scenario: 关键操作可用
- **GIVEN** 移动端触摸环境（无 hover）
- **WHEN** 玩家需要查看卡牌详情或执行核心操作
- **THEN** 不应依赖 hover 才能完成

### Requirement: 横竖屏布局切换
对局页 MUST 明确支持横竖屏切换：

- 同一套断点与方向规则在全项目一致
- 横屏下允许采用“左右分栏”，竖屏采用“上下分区”

#### Scenario: 横屏布局稳定
- **GIVEN** viewport 宽 >= 800 且高 <= 420（典型手机横屏紧凑高度）
- **WHEN** 进入对局页
- **THEN** 页面不出现明显横向溢出
- **AND** 战场与手牌区仍可见/可操作

### Requirement: 资源分级加载（移动端预算）
项目 MUST 支持资源分级加载策略，以满足移动端首屏性能：

- 首屏仅加载“立即可见 + 立即可交互”所需资源
- 非关键资源（大图、音频、复杂动效、教程重资源）应延迟加载

#### Scenario: 移动端首屏不被非关键资源阻塞
- **GIVEN** 移动网络/低端设备场景
- **WHEN** 进入对局页
- **THEN** 用户可以在首屏完成基本操作
- **AND** 非关键资源的加载失败不应导致白屏

### Requirement: 首屏速度优先（P0 可交互门禁）
移动端 MUST 优先保证“首屏可交互”，即使需要牺牲部分非关键视觉质量：

- 对局页进入后，P0 资源加载失败不应白屏（允许使用占位/降级背景）
- P1/P2 资源 MUST 延迟加载，不得阻塞首屏交互

#### Scenario: 首屏可操作优先于高清资源
- **GIVEN** 移动网络或 WebView 环境
- **WHEN** 进入对局页
- **THEN** 用户应能在首屏完成基础操作（如查看战场/打开手牌/点击主要按钮）
- **AND** 高清卡图/音效/特效允许在首屏后逐步补齐

### Requirement: 滑动/拖拽流畅度优先（输入与渲染预算）
移动端 MUST 优先保证手势交互的稳定帧率与低延迟：

- 拖拽/滑动交互期间 MUST 避免触发大面积 layout/reflow
- 动画与特效 MUST 遵守移动端预算，避免在低端机上造成明显掉帧

#### Scenario: 交互期间不启动重资源批量加载
- **GIVEN** 玩家正在进行滑动或拖拽手势
- **WHEN** 系统准备启动 P1/P2 资源批量加载
- **THEN** 加载任务应延后或降低并发，避免造成明显掉帧
