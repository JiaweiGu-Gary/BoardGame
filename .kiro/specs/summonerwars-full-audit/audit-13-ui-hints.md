# 审计 13 - UI 交互提示友好性审查

## 审查方法

逐个检查所有需要玩家交互的高风险能力，验证：
1. StatusBanners 中有对应的提示文本
2. 提示文本清晰描述操作要求
3. 有适当的确认/取消/跳过按钮
4. i18n 条目完整（zh-CN）

## 结论

✅ 所有 30+ 个高风险交互能力均有完整的 UI 提示，无缺失。

系统设计良好：
- StatusBanners 组件统一管理所有交互模式的提示横幅
- 每个模式有独立的颜色主题（amber/rose/cyan/indigo/teal/emerald/lime/sky/pink/orange/yellow）便于区分
- 资源不足时按钮自动 disabled + title 提示原因
- 多步交互有清晰的步骤提示
- "可选"效果均有跳过/取消按钮
- i18n 条目完整，中文描述清晰
